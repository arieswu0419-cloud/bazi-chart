# Aries 八字分析網站

純前端靜態網站，學員輸入姓名／性別／出生日期時間，計算四柱、五行、十神、大運、流年，並可點選大運／流年查看對應神煞。帳號需管理者審核後才能使用。

版面、欄位順序與配色都比照 [metagroup-bazi-ming-juan](https://www.event.meta-academy.biz/metagroup-bazi-ming-juan) 實際取樣調整：四柱欄位為「時／日／月／年」由左到右，大運與流年也改成跟四柱一樣的寬表格（十神／天干／地支／藏干／長生／神煞），點選欄位標題整欄反白，由老到少排列，並自動依今天日期反白目前所在的大運／流年。主色為金色 `#C9B872`、頁面底色為米色 `#EBE7DB`，取樣自參考網站本身的顏色。命盤結果可一鍵匯出成 PDF（含 Aries LOGO，大運流年會另起一頁）。

## 檔案結構

```
index.html          主頁面（命盤分析，需登入且審核通過）
login.html          登入頁
signup.html         申請帳號頁
css/style.css       版面樣式（響應式：桌面／平板／手機）
js/firebase-config.js   Firebase 專案設定（需自行填入）
js/auth.js          登入／註冊／審核狀態檢查
js/bazi-engine.js    八字計算（四柱／五行／十神／大運／流年／神煞／格局）
js/app.js           頁面互動邏輯
firestore.rules     Firestore 安全規則
assets/logo.png      Aries LOGO
```

## 一、建立 Firebase 專案（帳號審核系統）

GitHub Pages 只能放靜態檔案，帳號密碼與審核狀態需要 Firebase 來處理，一樣是免費額度。

1. 前往 [Firebase Console](https://console.firebase.google.com/)，建立新專案。
2. 左側選單「Build > Authentication」→ 開始使用 → 登入方式選「電子郵件/密碼」→ 啟用。
3. 左側選單「Build > Firestore Database」→ 建立資料庫 → 選正式環境模式（production mode）、選一個離您近的地區（如 asia-east1）。
4. 進入 Firestore 的「規則」分頁，把本專案 `firestore.rules` 的內容整份貼上取代，然後發布。
5. 左側選單「專案設定」（齒輪圖示）→ 一路往下滑到「您的應用程式」→ 選網頁圖示 `</>` 新增網頁應用程式 → 註冊後會看到一段 `firebaseConfig = {...}` 設定值。
6. 把這段設定值複製貼到 `js/firebase-config.js`，取代裡面的 `YOUR_API_KEY` 等預設文字。

## 二、審核學員帳號（管理者操作）

不用寫管理後台，直接用 Firebase 主控台：

- **核准新申請**：Firestore Database → `users` collection → 找到該學員的文件 → 把 `status` 欄位從 `"pending"` 改成 `"approved"` → 存檔。學員之後登入就能進入命盤分析頁。
- **停用某帳號**：Authentication 分頁 → 找到該使用者 → 右側選單「停用帳戶」。停用後該帳號即使密碼正確也無法登入。
- **第一個管理者/測試帳號**：請先自己到 `signup.html` 申請一個帳號，再回 Firestore 手動把自己的 `status` 改成 `approved`，這樣才能登入測試。

## 三、部署到 GitHub Pages

1. 在 GitHub 建立一個新的 repository（例如 `aries-bazi`）。
2. 把這個資料夾整包推上去：
   ```
   git init
   git add .
   git commit -m "Aries 八字分析網站"
   git branch -M main
   git remote add origin https://github.com/你的帳號/aries-bazi.git
   git push -u origin main
   ```
3. GitHub 上該 repo 的 Settings → Pages → Source 選 `main` 分支、`/ (root)` 目錄 → Save。
4. 幾分鐘後會產生網址，例如 `https://你的帳號.github.io/aries-bazi/`。

## 四、本機測試

直接雙擊 `index.html` 用瀏覽器打開即可（Firebase 設定填好之後）。若遇到瀏覽器安全限制導致 Firebase 連不上，可以改用簡易本機伺服器再開啟，例如：

```
python -m http.server 8000
```
再到瀏覽器打開 `http://localhost:8000/login.html`。

## 與參考網站／課程講義的比對結果

除了用瀏覽器實際操作 [metagroup-bazi-ming-juan](https://www.event.meta-academy.biz/metagroup-bazi-ming-juan) 背後的計算頁面逐項核對，也讀了 `補充/` 資料夾裡的課程講義（Meta Academy《八字命卷》系列 PDF），裡面第二卷第五章「十神命格判斷法」「十神特別格局」直接給了命格判斷的具體規則與範例，據此重寫了格局判斷邏輯；五行強弱表／十神比重表的加權公式則是拿好幾組不同生日的數字回推校正出來的（課程講義沒有寫出明確公式）。

**已逐項核對一致**：
- 四柱天干地支、五行、陰陽正負號
- 藏干內容與**顯示順序**（12 個地支的實際排列順序逐一測出來寫成對照表，不是猜的公式——多數地支是 lunar-javascript 原生順序反過來，但巳／未／申三支是另一種排法）
- 十神（天干、藏干）、十二長生
- **五行強弱表**：公式是「4 個天干 + 4 個地支的本氣」各算 1 分，中氣／餘氣不計分，總分 8，跟參考網站測試的多組數字完全一致
- **十神比重表**：天干（不含日主自己）算 1 分，地支本氣算 0.63 分、中氣／餘氣各算 0.2 分，子午卯酉這種只有一個藏干的地支算 1 分（等同天干）——這組權重是回推校正出來的，跟參考網站數字非常接近（多數落在四捨五入誤差內）
- **命格判斷**：改成課程講義教的邏輯——月支十二長生為「臨官」判建祿格、「帝旺」判羊刃格；否則看月支藏干本氣是否透干（出現在任一天干，含日干月干自己），本氣不透才看中氣／餘氣。多組測試案例（含課程講義本身的範例）判斷結果一致
- 天干合／克、地支三合／三會／六合／六沖的文字提示
- 生肖與年柱都用「立春」交界（精確到分鐘）判斷——測試了 2000 年立春交界前後（20:30 vs 20:45），年柱和生肖確實剛好在交界那一刻切換
- **大運／流年歲數**：改成跟參考網站一樣的「西曆實歲」（出生那一年＝0歲），原本用的是虛歲（出生那一年＝1歲），已修正並對照多組流年年份確認一致；大運也修正為固定抓 10 組（原本少抓了 2 組最年長的）
- 大運／流年現在跟四柱一樣，點開會顯示完整的十神／天干／地支／藏干／長生／神煞

**神煞（第二輪比對，讀了 `再補充/` 資料夾《八字解鎖篇》第十六、十七章的神煞專章後全部重寫）**：

原本的神煞是自己拼湊的坊間版本，跟參考網站對不上（例如月柱／年柱神煞完全不同）。《八字解鎖篇》第十七章逐頁列出每個神煞的正式對照表（天乙貴人、太極貴人、福星貴人、文昌貴人、天官貴人、國印貴人、學堂貴人、天德貴人、月德貴人、龍德、紅鸞天喜、桃花咸池、紅艷煞、孤辰寡宿、流霞、羊刃飛刃、血刃、劫煞、災煞、亡神、歲破大耗、五鬼、白虎、天狗、喪門、病符、天醫、金輿、祿神、魁罡貴人、十靈、陰差陽錯），照著逐條抄錄重寫後，用 1976-04-19 20:20（女）這組資料實測，**四柱每一柱的神煞內容都跟參考網站逐字一致**。過程中也發現並修正了幾個重要問題：

- **魁罡是「四柱有即算」**：原本誤以為只檢查日柱，實際上時柱／月柱／年柱只要干支剛好是庚辰、庚戌、壬辰、戊戌都算，不限日柱。
- **福星貴人有些天干對應兩個地支**（乙對應丑、亥；丙對應子、戌），原本只寫了一個，漏掉一半。
- **華蓋、將星等三合局神煞會有「自己對自己」的假命中**：地支剛好落在辰戌丑未（庫位）時，用該支自己的三合局去查，會查到自己，這種情況不算數，已修正排除。
- **年空／空亡的顯示**：四柱的地支旁邊會分別對照「日柱所在旬」與「年柱所在旬」，命中哪一組空亡就標示「空」或「年空」，現在跟參考網站一樣會顯示出來。

**已知還沒解決的一個案例**：測試 1990-05-18 14:30（男）這組資料時，命格算出「正印格」，但參考網站顯示「正官格」——月支巳的藏干本氣（丙）不透干，中氣（庚）有透干（年干庚），照課程講義規則應該用中氣定格（正印格才對），但參考網站卻是用餘氣（戊）判定為正官格。已經深入比對多組資料、也讀了課程講義相關章節，還是沒找到能同時解釋這個案例又不跟其他案例衝突的規則，可能牽涉到更進階的判斷（例如身強身弱的取捨），暫時保留這個已知差異，其餘測試案例的命格都判斷正確。

## 其他已知限制

- **四柱提示**：只做天干合／克（相鄰兩柱）與地支三合／三會／六合／六沖偵測，**沒有判斷「化／不化」**，也還沒做「刑」「破」「害」偵測（1976-04-19 這組測試資料，參考網站地支提示顯示「戌丑刑,辰辰刑,辰丑破」，目前網站只會顯示「無明顯合沖會局」，這部分還沒實作）。
- **課程資料連結功能已依需求移除**，網站現在只做命盤分析。
